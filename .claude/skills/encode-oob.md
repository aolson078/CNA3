# Encode OOB

Structured Order of Battle data entry for CNA scenarios.

## Input

The user provides a scenario identifier, e.g., `compass_axis`, `crusader`, `battleaxe`.

## Unit Schema

Each unit entry contains:

```python
UnitEntry(
    designation: str,       # e.g., "7th Armoured Division"
    side: str,              # "allied" or "axis"
    nationality: str,       # "british", "italian", "german", "australian", etc.
    unit_type: str,         # see valid types below
    toe_strength: int,      # Table of Equipment strength rating
    cpa: int,               # Capability Point Allowance per turn
    stacking_points: int,   # stacking value (determines how many fit in a hex)
    movement_allowance: int, # base movement points per turn
    attack_strength: int,   # base attack factor
    defense_strength: int,  # base defense factor
    special: list[str],     # special capabilities (see below)
    hex_placement: str,     # starting hex ID, e.g., "C2010"
    parent_formation: str,  # parent unit designation, if any
    notes: str,             # free-text notes from the OOB chart
)
```

### Valid Unit Types
- `infantry`, `armour`, `motorized`, `mechanized`
- `artillery`, `anti_tank`, `anti_air`
- `recon`, `engineer`, `headquarters`
- `supply`, `transport`
- `garrison`, `fortress`
- `air_unit` (handled separately, but can appear in OOB)

### Valid Special Capabilities
- `elite`, `green`, `motorized_movement`, `desert_trained`
- `heavy_artillery`, `self_propelled`, `long_range`
- `bridging`, `mine_clearing`, `commando`

## Procedure

### Step 1 --- Set up the scenario

1. Read the scenario identifier from the user's input.
2. Check if `cna/data/oob/{scenario}.py` already exists --- if so, read it and report the current unit count, offering to continue or start fresh.
3. Ask the user which side they're entering first (`allied` or `axis`).

### Step 2 --- Guided unit entry

For each unit, prompt through the fields:

1. **Designation**: Free text, e.g., "7th Armoured Division / 4th Armoured Brigade"
2. **Side**: Pre-filled from Step 1 selection
3. **Nationality**: Prompt with common options for the selected side
4. **Unit type**: Prompt with valid types list
5. **TOE strength**: Integer --- warn if outside typical range (1-20)
6. **CPA**: Integer --- warn if outside typical range (1-15)
7. **Stacking points**: Integer --- warn if outside typical range (1-6)
8. **Movement allowance**: Integer --- warn if outside typical range for the unit type:
   - Infantry: 3-6
   - Armour/Motorized: 8-16
   - Artillery: 4-8
9. **Attack / Defense strength**: Integers
10. **Special capabilities**: Comma-separated from the valid list, or empty
11. **Hex placement**: Hex ID --- validate format
12. **Parent formation**: Free text or empty
13. **Notes**: Free text or empty

The user can also enter units in a compact one-line format:
```
"7th Armoured / 4th Bde | british armour | 12/8/4 | CPA 10 | SP 3 | MA 12 | elite,desert_trained | C2010"
```
Parse this and confirm the interpreted values.

### Step 3 --- Validation

For every unit entry, validate:

1. **Hex format**: Must match `[A-Z]\d{4}`
2. **Stat sanity checks**:
   - Attack > Defense for offensive units (armour, recon) --- warn if reversed (may be intentional)
   - CPA of 0 is suspicious --- confirm with user
   - Stacking points of 0 is invalid
3. **Type-stat consistency**:
   - Artillery should not have high movement allowance (>8) unless `self_propelled`
   - Infantry should not have movement allowance >8 unless `motorized_movement`
   - HQ units typically have low attack/defense
4. **Duplicate detection**: Warn if a unit with the same designation already exists
5. **Formation grouping**: If parent_formation is set, check that the parent exists or will exist

### Step 4 --- Output

Write to `cna/data/oob/{scenario}.py`:

```python
"""Order of Battle --- {Scenario Name}

Auto-generated by /encode-oob. Do not edit manually.
"""

ALLIED_OOB = [
    {
        "designation": "7th Armoured Division / 4th Armoured Brigade",
        "side": "allied",
        "nationality": "british",
        "unit_type": "armour",
        "toe_strength": 12,
        "cpa": 10,
        "stacking_points": 3,
        "movement_allowance": 12,
        "attack_strength": 8,
        "defense_strength": 4,
        "special": ["elite", "desert_trained"],
        "hex_placement": "C2010",
        "parent_formation": "7th Armoured Division",
        "notes": "",
    },
    # ... more units
]

AXIS_OOB = [
    # ... axis units
]
```

### Step 5 --- Summary

After each entry session, report:
- Total units entered (allied / axis breakdown)
- Any validation warnings that were overridden
- Formation hierarchy summary (which parent formations and their sub-units)
